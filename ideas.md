# Chosen Direction

I am choosing **Scholarly Marginalia**.

This direction best fits the user's request for something minimal and self-contained while still making the page feel intentional rather than generic. It allows the site to read as an academic calling card: one page, text-led, calm, and structured around clear exits to LinkedIn and GitHub. Because the user identifies first as a mathematician and applied computing student, the homepage should avoid startup-style portfolio language and instead present a concise scholarly identity, current affiliations, and selected work with editorial restraint.

The resulting homepage should therefore have four compact zones: a strong introduction, a brief academic profile, a link rail for LinkedIn and GitHub, and a selected projects section pointing to pinned repositories. The design philosophy to test throughout implementation is simple: **does this choice reinforce intellectual clarity and quiet authority, or does it dilute it?**
